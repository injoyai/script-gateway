网关系统需求
一、总体目标
构建脚本驱动的通用网关，核心环节（监听、处理、分发）均可通过 Golang 脚本（基于 yaegi 解释器）深度定制。提供现代化前端（Vue/React），支持实时监控、配置管理与基础认证。

yaegi 定位说明：脚本仅用于中等及以下复杂度的定制逻辑（格式转换、简单路由、字段映射等）。高吞吐数据处理应优先使用内置处理器。yaegi 脚本性能约为基础 Go 代码的 5~10 倍降幅，单节点仍可支撑 10k msg/s 前提是脚本内无重计算。

二、核心特性（优化后）
脚本化：监听、处理器、分发器均可用 yaegi 脚本替代默认逻辑。脚本可访问上下文（原始数据、处理后数据、元信息、配置对象）。

内置编辑器：集成 Monaco Editor，提供语法高亮、基础补全；后端预编译脚本进行错误检查。

多协议监听：内置 TCP/UDP/MQTT/WebSocket/Serial（串口），支持脚本自定义协议。

管道式数据处理：一个监听器数据可按顺序经过多个处理器（内置/脚本），最后推入内部 Go channel 队列（无锁，高性能）。水平扩展时可选切换为 Redis Stream 或 NATS。

灵活分发：内置分发器（DB、MQ、HTTP 回调）+ 脚本自定义分发。分发器订阅队列 topic（topic 由监听器配置的 output_topic 或处理器链脚本动态指定）。

可观测性：前端展示监听器状态、吞吐量、错误率（解析错误/脚本错误/分发失败）；历史图表（折线图/柱状图）。

配置管理：统一存于数据库（MySQL/PostgreSQL 生产，SQLite 开发），热加载（新配置仅对新连接/新请求生效）。提供配置快照回滚（保留最近3份 JSON 全量快照）。

用户认证：单用户模式（admin/admin），密码 bcrypt 存储，JWT 认证。增加操作日志表记录所有配置变更。

三、详细功能模块（针对 yaegi 的适配优化）
3.1 脚本引擎（基于 yaegi）
使用 yaegi 解释执行 Go 脚本。

性能补偿措施：

脚本运行超时限制（默认 50ms），超时后中断并记录错误。

脚本结果缓存（相同输入+相同脚本版本可复用输出，需用户标记幂等）。

脚本内禁止复杂循环和递归（代码静态检查警告）。

安全沙盒：

编译时通过 Interpreter 的 Use 方法仅允许导入白名单包（如 strings、strconv、encoding/json、自定义辅助库）。

使用 yaegi 的 Symbol 过滤，禁止 os/exec、syscall、net 中除 HTTP 客户端外的包。

运行时注入受限的 http.Client（带超时、禁用文件访问）。

脚本接口：预定义 ScriptProcessor、ScriptListener、ScriptDispatcher 接口，脚本需实现对应方法。

3.2 监听器模块

先定义好统一的监听器接口,包括监听,返回统一接口,包括 开始监听,停止监听,获取监听状态。

脚本自定义监听器：需实现监听器接口

开始监听: Run()error
停止监听: Close()error
获取监听状态: Closed()bool
处理数据: OnConnect(conn io.ReadWriteCloser)//大概这样,如果不合理可以进行修改,需要一个统一接口,能兼容锁头监听类型

3.3 数据处理管道
处理器链顺序执行，每个处理器可返回错误中断链路。

最后输出必须携带 topic 字段（可由脚本动态指定或使用监听器默认的 output_topic）。

输出推送到内部队列（chan *Message），消息包含 Payload、Topic、Metadata（来源监听器ID、时间戳、链路ID）。

3.4 分发器模块
多个分发器可订阅不同 topic，每个分发器独立配置目标类型和参数。

可选关联脚本，用于动态修改目标地址、数据格式或完全自定义分发逻辑。

内置分发器：MySQL/PostgreSQL（支持 upsert）、Kafka（Sarama 客户端）、RabbitMQ、HTTP 回调,os.Stdout等。

3.5 后端 API（Gin + Gorm）
认证：POST /api/login、POST /api/logout、PUT /api/user/password

配置 CRUD：监听器、处理器链、分发器、脚本

监控：

GET /api/metrics（实时指标：瞬时速率、错误计数、连接数）

GET /api/history?metric=xxx&start=...&end=...&resolution=1m（查询时序库）

操作日志：GET /api/audit（仅管理员）

快照管理：POST /api/snapshot（手动创建快照）、POST /api/rollback（恢复指定快照）

3.6 数据库设计（生产使用 MySQL/sqlite）

3.7 前端页面（菜单结构优化）
仪表盘：KPI卡片、折线图（速率/错误率）、监听器状态列表

监听管理：增删改查内置监听器（TCP/UDP/MQTT/WebSocket/Serial）或脚本监听器；配置 output_topic。

采集管理（预留）

推送管理：分发器管理，支持内置分发器或脚本分发器；可配置定时/实时（实时即立即消费）

消息队列：展示内部队列深度、积压情况；若启用外部队列（Redis/NATS），可显示其状态。

配置管理：统一管理处理器链、脚本文件（按类型分目录展示）、全局快照及回滚。

全局设置：修改密码、操作日志查看、系统快照管理。

四、非功能需求（适配 yaegi）
性能：

单节点 ≥10k msg/s（脚本为简单字段映射，平均执行时间 <10µs）。若脚本复杂度较高，吞吐会下降。

内部队列使用无锁环形队列（github.com/zeromicro/go-queue 或自研）。

可靠性：

脚本 panic 被 recover，记录错误并继续处理下一条消息（避免阻塞整个管道）。

监听器支持自动重连（客户端型协议）。

安全性：

API 全部鉴权（除 /api/login）。

密码 bcrypt 哈希，JWT 过期时间 24h。

可扩展性：

脚本热加载：编辑保存后重新解析 yaegi 符号，下次调用时生效（无需重启）。

水平扩展：要求使用外部消息队列（Redis Stream / NATS）和 MySQL/PostgreSQL，实例间通过队列共享负载。

部署：

单一二进制后端 + 前端静态文件。

提供 docker-compose.prod.yml（网关 + MySQL + Redis + Prometheus 可选）和 docker-compose.dev.yml（网关 + SQLite）。
